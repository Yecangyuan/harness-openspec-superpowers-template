# Compound-Knowledge Workflow

Use this workflow after valuable project-specific learning should be preserved.

## Inputs

- The problem or decision just resolved
- Final implementation or design choice
- Failed attempts or rejected alternatives
- Existing `knowledge/index.md`, when present

## Steps

1. Read `harness/skills/compound-knowledge/SKILL.md`.
2. Create `knowledge/` with standard subdirectories if needed.
3. Write one focused topic file under the right category.
4. Add one concise line to `knowledge/index.md`.
5. Remove or update stale index entries that contradict the new learning.

## Output

One new or updated topic file plus an updated lightweight index.
